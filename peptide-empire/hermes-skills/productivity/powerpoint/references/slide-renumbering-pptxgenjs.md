# Slide Renumbering When Inserting Slides (PptxGenJS)

## Problem

When inserting a new slide into a PptxGenJS-generated deck, all `addSlideNum(s, N)` calls after the insertion point must shift by +1. Doing this manually via patch is error-prone — duplicate numbers are easy to miss, and working backwards from the highest number to avoid collisions is tedious.

## Solution: Python script to programmatically shift

### Step 1: Insert the new slide code in the JS

Add the new slide's code block with its correct `addSlideNum()` at the insertion point. Example: inserting a hero slide at position 2 means the new slide gets `addSlideNum(s, 2)`.

### Step 2: Run the renumbering script

```python
import re

with open('deck.js', 'r') as f:
    content = f.read()

lines = content.split('\n')
changes = []

# Find all addSlideNum calls with line numbers
for i, line in enumerate(lines):
    m = re.search(r'addSlideNum\(s, (\d+)\)', line)
    if m:
        changes.append((i, int(m.group(1))))

# Identify which occurrence is the newly inserted slide
# (the one whose context includes the insertion marker)
# Example: if inserting after "HERO (CYBERCAB", skip that one
for line_no, old_num in reversed(changes):
    context_start = max(0, line_no - 5)
    context = '\n'.join(lines[context_start:line_no])
    is_new_slide = 'HERO (CYBERCAB' in context  # match your insertion marker
    
    if is_new_slide:
        continue  # New slide keeps its number
    
    new_num = old_num + 1
    old_text = f'addSlideNum(s, {old_num})'
    new_text = f'addSlideNum(s, {new_num})'
    lines[line_no] = lines[line_no].replace(old_text, new_text)

with open('deck.js', 'w') as f:
    f.write('\n'.join(lines))

# Verify: no duplicates, sequential range
nums = [int(x) for x in re.findall(r'addSlideNum\(s, (\d+)\)', '\n'.join(lines))]
from collections import Counter
dupes = [n for n, c in Counter(nums).items() if c > 1]
if dupes:
    print(f"WARNING: Duplicates: {dupes}")
else:
    print(f"OK: {len(nums)} slides, range {min(nums)}–{max(nums)}")
```

### Step 3: Update total page count

Change the `/ N` denominator in the `addSlideNum` function:

```javascript
// Before
slide.addText(`${num} / 16`, { ... });

// After
slide.addText(`${num} / 17`, { ... });
```

### Step 4: Verify

```python
import re
nums = [int(x) for x in re.findall(r'addSlideNum\(s, (\d+)\)', content)]
print(f"Slide range: {min(nums)}–{max(nums)}, count: {len(nums)}")
# Should be: range 1–N, count = N
```

## Pitfall: The insertion marker

The `is_new_slide` check depends on finding unique context near the newly inserted slide. Make sure the marker string (e.g., `'HERO (CYBERCAB'`) appears only in the new slide's code block. If it appears elsewhere, the script will skip the wrong occurrence.

## When NOT to use

- If the deck was not generated by PptxGenJS and doesn't use `addSlideNum()` — the XML-based slide numbering in template decks is handled differently
- If you're only editing text, not adding/removing slides — no renumbering needed
