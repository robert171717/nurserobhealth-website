# PDF Guide Generator

**Purpose:** Create professional, branded multi-page PDFs programmatically from structured content — guides, reports, lead magnets, ebooks — using Python's fpdf2 OR reportlab library depending on complexity needs. Use reportlab for complex layouts (full-width colored section bars, rounded-corner boxes, precise page-break control). Use fpdf2 for simpler guides where TTF font handling is more important than visual polish.

**Wrapping tables** (use when cells contain long text that needs to wrap within columns):
```python
def wrapping_table_hdr(cols, widths, row_h=7):
    pdf.set_fill_color(10, 31, 63)
    pdf.set_text_color(255, 255, 255)
    pdf.set_font('D', 'B', 8)
    x_start = pdf.get_x()
    y_start = pdf.get_y()
    max_y = y_start
    for i, col in enumerate(cols):
        x_pos = x_start + sum(widths[:i])
        pdf.set_xy(x_pos, y_start)
        pdf.multi_cell(widths[i], row_h/2, ' ' + col, border=1, fill=True, align='C')
        max_y = max(max_y, pdf.get_y())
    pdf.set_y(max_y)

def wrapping_row(cells, widths, row_h=6, fill=False):
    pdf.set_fill_color(247, 249, 251) if fill else pdf.set_fill_color(255, 255, 255)
    pdf.set_text_color(30, 41, 59)
    pdf.set_font('D', '', 7.5)
    x_start = pdf.get_x()
    y_start = pdf.get_y()
    max_y = y_start
    for i, cell in enumerate(cells):
        x_pos = x_start + sum(widths[:i])
        pdf.set_xy(x_pos, y_start)
        pdf.multi_cell(widths[i], row_h/2, ' ' + cell, border=1, fill=True, align='L')
        max_y = max(max_y, pdf.get_y())
    pdf.set_y(max_y)
```

**When to use wrapping tables:** Any table where a column cell is longer than ~15 characters. The wrapping approach uses `multi_cell` with precise x/y positioning so text stays in its column box. Adjust `row_h` per row — use 8-10 for single-line cells, 12-14 for cells with multiple sentences. Left-align (`align='L'`) wrapped cell text for readability; keep headers centered (`align='C'`).

### Key Differences from fpdf2
- **Platypus story-based**: Build a Python list of flowables (Paragraph, Table, Spacer, PageBreak), then `doc.build(story)`. Each `.build()` call consumes the story — build once and copy the output file for backups.
- **Styles are ParagraphStyle objects**: Define once, reuse. `fontName='Helvetica-Bold'`, `fontSize=10`, `leading=15`.
- **Page breaks are explicit**: `story.append(PageBreak())` before each new section. This is the main reason to use reportlab over fpdf2 for guides with distinct sections.
- **Full-width section headers**: Use a single-cell Table with navy background spanning the page width.
- **Headers/footers**: Use `doc.build(story, onFirstPage=header_fn, onLaterPages=header_fn)` with canvas drawing functions.

### Reportlab Pitfalls
- **`.build()` consumes the story list** — you cannot call `.build()` twice on the same story. Build once, then copy the output file to other locations with `shutil.copy()`.
- **Table cells need Paragraph objects for wrapped text** — plain strings in cells get truncated. Use `Paragraph(text, style)` for any cell with multi-line content.
- **`ROUNDEDCORNERS` requires reportlab ≥4.0** — it's available in the Hermes venv.
- **reportlab is NOT pre-installed** — install with `pip3 install reportlab` using the Hermes venv pip. fpdf2 IS pre-installed.
- **Page breaks before sections**: Always `story.append(PageBreak())` before each `story.extend(section_start(...))` call. This guarantees sections start at page tops.

### Font Encoding Errors
`FPDFUnicodeEncodingException` — the built-in Helvetica font doesn't support Unicode characters (em dashes, smart quotes, certain symbols).
**Fix:** Always use DejaVu Sans TTF (add_font with custom .ttf path). Never use built-in Helvetica/Times/Courier for content with special characters.

### multi_cell Position Errors
`Not enough horizontal space to render a single character` — x position is stuck from a previous cell() call.
**Fix:** Call `pdf.set_x(pdf.l_margin)` before every `multi_cell()` to reset horizontal position.

### Deprecation Warnings (fpdf2 v2.5+)
- `ln=True` deprecated → use `new_x="LMARGIN", new_y="NEXT"`
- `uni=True` deprecated in add_font → just omit it

### Table Header Contrast
Dark backgrounds (navy) need WHITE text. Always use `pdf.set_text_color(255, 255, 255)` before table headers.
Light backgrounds need dark text. Use `pdf.set_text_color(30, 41, 59)` for table rows.

### Footer on Cover Page
Set up `pdf.header` and `pdf.footer` AFTER the cover page, or check `pdf.page_no() > 1` in the header function.

### {nb} Not Working in Footer
If `{nb}` (total page count) shows as literal text instead of the number, you forgot to call `pdf.alias_nb_pages()` after creating the PDF instance. Add it right after `pdf = FPDF()`.
